Openbentt — Desktop System Design

Scope: Electron desktop only. Product: Openbentt (local-first chat + OpenRouter + workspaces).

1. Problem solved

  • Privacy / offline-capable AI: On-device inference via WebGPU/WASM (Transformers.js in the renderer) or native GGUF via a local llama-server child process.
  • One UI everywhere: Same Vite/React app; Electron adds shell, IPC, and Chromium flags.
  • GGUF without Ollama: Main process downloads .gguf, manages registry under userData, spawns OpenAI-compatible server on 127.0.0.1.
  • Linux WebGPU: Early app.commandLine.appendSwitch calls enable WebGPU paths that often fail in stock browser profiles.
  • HF token handling: safeStorage-backed (or restricted fallback) storage for gated model downloads.

4. Distribution plan

  • Trigger: Git tag v* → .github/workflows/release.yml.
  • Outputs: Linux AppImage + deb; Windows NSIS exe (+ zip); macOS dmg + zip; optional static web zip.
  • Build: npm run build then electron-builder per OS; BusyTeX fetched for LaTeX features.
  • Signing: Documented as optional; unsigned builds may trigger Gatekeeper / SmartScreen.
  • Note: Shipping llama-server in installers may require extraResources — today users often rely on PATH or manual path.

5. Improvements (desktop)

  • Bundled per-platform llama-server in releases.
  • macOS notarization + Windows code signing.
  • Auto-update channel (e.g. electron-updater).
  • Strict IPC payload validation; single-instance lock.
  • Opt-in crash reporting (no chat content).
