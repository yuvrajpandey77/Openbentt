# Openbentt v2.2.3 — Universal Linux GPU compatibility

Production-ready desktop release with **automatic software rendering** on broken or missing GPU drivers (NVIDIA + Wayland, headless, no DRI). The window opens reliably; menus use the **native system title bar** for maximum workspace.

## Download

| Platform | File | Install |
|----------|------|---------|
| **Linux (recommended)** | `openbentt_2.2.3_amd64.deb` | `sudo dpkg -i openbentt_2.2.3_amd64.deb` then run `openbentt` |
| **Linux (portable)** | `Openbentt-2.2.3.AppImage` | `chmod +x Openbentt-2.2.3.AppImage && ./Openbentt-2.2.3.AppImage` |
| **Windows** | `Openbentt Setup 2.2.3.exe` | Run installer (unsigned — SmartScreen may warn) |
| **macOS** | `Openbentt-2.2.3.dmg` | Open DMG, drag to Applications (unsigned — Gatekeeper may warn) |
| **Web (self-host)** | `openbentt-web-dist.zip` | Unzip `dist/` to any static host with SPA fallback |

Desktop builds include **llama-server** for optional on-device GGUF models. Web zip is OpenRouter + Notebook only (no Electron shell).

## Quick start (desktop)

1. **Install** using the table above.
2. **Launch** Openbentt — it opens on the **Projects** hub.
3. **OpenRouter API key** (required for cloud AI): click the banner on Projects or go to **Settings → API**. Free models are available at [openrouter.ai/keys](https://openrouter.ai/keys).
4. **Create a project** → open **Notebook Studio** to upload PDFs, write LaTeX, and chat with your corpus.
5. **Knowledge panel** (Notebook left rail → Knowledge): persistent project memory injected into every chat; use **AI update** to synthesize from papers + draft.
6. **Updates**: **Settings → General → Check for updates** (uses these GitHub Releases).

## Linux GPU / display (read this if the app misbehaves)

Openbentt **auto-detects** these cases and enables **software rendering** (no GPU required):

- NVIDIA GPU on **Wayland** (common on Ubuntu/Kali laptops)
- NVIDIA GPU with **no proprietary driver** loaded (`driver (null)` in logs)
- No `/dev/dri` GPU device
- `LIBGL_ALWAYS_SOFTWARE=1` or Mesa software drivers

When safe mode is active you will see in the terminal:

```text
[electron] Software rendering enabled (nvidia-on-wayland|nvidia-no-driver|…)
[electron] Software rendering: ozone-platform=wayland (native, not XWayland)
```

**UI in safe mode:** standard window frame with **File / Edit / View / Help** in the **system menu bar** (not the custom in-app title strip) — more vertical space for your work. Window icon shows the Openbentt logo.

**Harmless startup lines** (ignore): `GBM-DRV error`, `pci id … driver (null)`, `SQLite is an experimental feature`.

### Environment overrides

| Variable | Effect |
|----------|--------|
| `OPENBENTT_DISABLE_GPU=1` | Force software rendering |
| `OPENBENTT_DISABLE_GPU=0` | Force hardware GPU (only if drivers work) |
| `OPENBENTT_OZONE_PLATFORM=wayland\|x11\|auto` | Override display backend |

**If a previous instance is stuck:** `pkill -9 -f openbentt` then relaunch.

## What’s new since v2.2.0

### Compatibility (2.2.1–2.2.3)

- Native **Wayland** + software rasterizer instead of XWayland (fixes invisible window on NVIDIA laptops)
- **NVIDIA PCI detection** without requiring proprietary driver files
- **Framed window** + visible **native menu bar** in safe mode
- Aggressive window show/focus fallbacks so production never stays icon-only

### Research & UX (from v2.2.0)

- **Knowledge** panel — persistent project context for AI
- **Auto-RAG** on every chat message when PDFs are loaded
- **Chat logs** persisted per project in SQLite
- **Tools** panel inline in Notebook explorer (VS Code–style)
- **Tools dialog** in chat (centered, organized)

## Requirements

- **Desktop:** Linux x64 (glibc), Windows 10+, or macOS 11+
- **Web / API:** OpenRouter API key for cloud models
- **Optional local models:** GGUF files + llama-server (bundled in desktop installers)

## Security note

Linux/Windows/macOS installers are **not code-signed**. You may need to allow the app in your OS security dialog. Keys stay **local** on your machine ([THREAT_MODEL](https://github.com/yuvrajpandey77/Openbentt/blob/main/docs/THREAT_MODEL.md)).
