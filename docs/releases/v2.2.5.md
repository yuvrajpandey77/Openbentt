# Openbentt v2.2.5 — Download fixes + release pipeline repair

This patch release restores reliable **direct download** links on the site and fixes a broken production build introduced during the v2.2.4 merge.

## Installers

| Platform | File | Notes |
|----------|------|-------|
| **Linux (recommended)** | `openbentt_2.2.5_amd64.deb` | `sudo dpkg -i openbentt_2.2.5_amd64.deb` |
| **Linux (portable)** | `Openbentt-2.2.5.AppImage` | `chmod +x Openbentt-2.2.5.AppImage && ./Openbentt-2.2.5.AppImage` |
| **Windows** | `Openbentt Setup 2.2.5.exe` | NSIS installer (unsigned — SmartScreen may warn) |
| **Windows (portable)** | `Openbentt.exe` | No installer |
| **Windows (zip)** | `Openbentt-2.2.5-win.zip` | Portable archive |
| **macOS (Apple Silicon)** | `Openbentt-2.2.5-arm64.dmg` | Unsigned — Gatekeeper may warn |
| **Web static bundle** | `openbentt-web-dist.zip` | Self-host the Vite build |

## What shipped

### 1) Direct download links restored

- Download buttons on `/download` and marketing CTAs again point at **GitHub asset URLs** (`browser_download_url` or `releases/latest/download/...`), not the GitHub Releases page.
- Fallback filenames are derived from the **latest release tag** so stale build-time version env vars cannot send users to old artifacts.

### 2) Production build fix

- Repaired `fetchLatestReleaseAssets.ts` after a bad merge (duplicate symbol + missing helper) that broke Vercel deploys.

### 3) Release alignment

- App version bumped to **2.2.5** across `package.json`, lockfile, and download fallbacks.
- CI continues to verify tag ↔ `package.json` before publishing installers.

## Validation

```bash
npm run lint
```

## Upgrade notes

- Desktop auto-update picks up this release when `latest*.yml` manifests are published by CI.
- Web deploy: merge to `main` after tagging; Vercel rebuilds with `VITE_APP_VERSION=2.2.5` from `package.json`.
